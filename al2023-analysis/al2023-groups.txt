System Base Groups:

1. Core - Absolute minimal (5 packages: basesystem, bash, ca-certificates, coreutils, system-release)
2. Container - For Docker/Podman containers
3. Base (minimal) - Minimal system installation
4. Base - Standard base system

Cloud/VM Image Groups:

5. AMI (minimal) - Minimal AWS EC2 image
6. AMI (minimal, kernel 6.12) - Minimal AWS EC2 with newer kernel
7. AMI - Full AWS EC2 image
8. AMI (kernel 6.12) - Full AWS EC2 with newer kernel
9. OnPrem (minimal) - Minimal on-premises/Hyper-V/VMware image (57 packages)
10. OnPrem - Full on-premises/Hyper-V/VMware image

Development/Tools Groups:
11. AWS Tools - AWS CLI and tools
12. Development Tools - Compilers, build tools (gcc, make, etc.)
13. buildsys-build - Package building tools
14. Desktop - Desktop environment packages

Example to explain different kinds of pkgs in a group


# Install only mandatory packages
dnf group install "Base" --no-default

# Install mandatory + default packages (standard behavior)
dnf group install "Base"

# Install mandatory + default + optional packages
dnf group install "Base" --with-optional

# Install mandatory + specific optional packages
dnf group install "Base"
dnf install acpid zsh  # install optional packages separately
